# Roma Aeterna 3.5.1 — Project Purchase Fix

Исправлена обработка муниципальных строительных проектов Opera Publica.

## Что исправлено

1. Если игрок выбирает вариант «построить из запасов», но материалов не хватает, игра больше не откладывает проект автоматически. При наличии денег она выбирает доступный вариант срочной закупки материалов.
2. Для `build_buy` проверяется только итоговая сумма золота. Остаточные требования ресурсов из старых сохранений больше не блокируют подряд.
3. Вместо общего сообщения «недостаточно средств или материалов» интерфейс показывает конкретно, чего не хватает: золота, зерна или определённого ресурса.
4. Если ни один строительный вариант действительно недоступен, проект по-прежнему корректно откладывается.
5. Версия игры обновлена до `3.5.1-project-purchase-fix`.

## Замена файлов

Скопируйте с заменой в корень проекта:

- `roma_aeterna.py`
- `roma_buildings.py`
- `roma_city_events.py`

Старые сохранения поддерживаются. Для уже сохранённых событий `build_buy` добавлена совместимость с устаревшим полем `requires.resources`.

## Проверки

- `py_compile` всех трёх файлов: успешно.
- Сценарий: материалов нет, вариант 1 недоступен, вариант 2 доступен — успешно.
- Автоматический переход с варианта 1 на срочную закупку — успешно.
- Исполнение `build_buy`, списание золота и завершение здания — успешно.
- Совместимость со старым событием, где `build_buy` ошибочно содержит требования ресурсов — успешно.
